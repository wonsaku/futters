# Futters 개발 안내

## 이 프로젝트는 무엇인가요?

풋살 경기 기록 및 통계 관리 서비스입니다.
- **Next.js** — 웹사이트 화면을 만드는 도구 (리액트 기반 프레임워크)
- **Supabase** — 데이터베이스 및 로그인 기능을 제공하는 서버 서비스
- **Vercel** — 만든 웹사이트를 인터넷에 올려주는 배포 서비스

---

## Claude와 협업하는 방식

### 기본 원칙
- **사용자가 요청한 것만** 단계적으로 수행합니다. 먼저 앞서 나가지 않습니다.
- 코드 작업, 파일 생성, 설정 변경 등 **자동으로 할 수 있는 건 최대한 직접 처리**합니다.
- 계정 생성, 비밀번호 입력, 결제 등 **사람이 직접 해야 하는 작업**은 단계별로 쉽게 안내합니다.

### 용어 설명 방식
개발 용어가 나올 때는 항상 괄호 안에 쉬운 설명을 함께 표시합니다.
예: `배포(deploy) — 만든 웹사이트를 인터넷에 올리는 것`

---

## 작업 완료 후 항상 표시할 진행 현황

각 작업이 끝날 때마다 아래와 같은 형식으로 진행 상황을 보여줍니다.

```
✅ 완료된 단계
🔄 지금 진행한 단계
⬜ 아직 남은 단계
🙋 사람이 직접 해야 할 작업 (있을 경우)
```

---

## 현재 프로젝트 구성

- **프레임워크**: Next.js (버전은 node_modules에서 확인)
- **데이터베이스**: Supabase (로그인 기능 포함)
- **배포 예정**: Vercel
- **개발 환경**: Windows 11, PowerShell

---

## 디자인 시스템

| 변수 | 색상 | 용도 |
|------|------|------|
| `--futters-green` | #16A34A | 주요 색상 (버튼, 강조) |
| `--futters-dark` | #14532D | 제목, 진한 텍스트 |
| `--futters-light` | #F0FDF4 | 배경 |
| `--futters-warm` | #DCFCE7 | 카드 배경 |
| `--futters-gray` | #6B7280 | 보조 텍스트 |
| `--futters-accent` | #F59E0B | 강조 포인트 |

공통 클래스: `futters-card`, `futters-input`, `futters-btn-primary`, `futters-gradient`

---

## 사람이 직접 해야 하는 대표적인 작업들

| 작업 | 이유 |
|------|------|
| Supabase 계정 생성 및 프로젝트 생성 | 개인 계정이 필요함 |
| .env.local에 환경변수 입력 | 보안상 직접 입력해야 함 |
| Vercel 계정 연결 및 배포 승인 | 개인 계정이 필요함 |

---

## Next.js 코드 작성 전 필독

코드를 작성하기 전에 반드시 `node_modules/next/dist/docs/` 폴더의 관련 가이드를 먼저 읽습니다.
학습 데이터와 실제 설치된 버전이 다를 수 있으므로, 항상 설치된 버전 기준으로 작업합니다.
